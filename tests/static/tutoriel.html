Tutorial: Checking broken links with GoLink

This tutorial will guide you step-by-step through using GoLink to check links in your documentation.

Step 1: Prepare your project

Create a configuration file golink.yml at the root of your project:

version: 1
output: rapport-liens.json
ignore:
  - "*.example.org"
  - "*.local"
depth: 3
timeout: 10

For more configuration options, check out our advanced configuration example (internal invalid link).

Step 2: Run GoLink

Execute the following command to analyze your documentation:

golink check --dir ./docs

You can also check a remote website:

golink check --url https://yoursite.com/docs

Step 3: Analyze the report

GoLink will generate a detailed report. To understand this report, consult our documentation on report format (external invalid link).

Step 4: Integrate with CI/CD

Integrate GoLink into your CI/CD pipeline by following our CI/CD integration guide.
