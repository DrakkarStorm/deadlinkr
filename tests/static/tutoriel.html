Tutoriel: Vérifier les liens morts avec GoLink

Ce tutoriel vous guidera étape par étape dans l'utilisation de GoLink pour vérifier les liens dans votre documentation.

Étape 1: Préparer votre projet

Créez un fichier de configuration golink.yml à la racine de votre projet :

version: 1
output: rapport-liens.json
ignore:
  - "*.example.org"
  - "*.local"
depth: 3
timeout: 10

Pour plus d'options de configuration, consultez notre exemple de configuration avancée (lien interne invalide).

Étape 2: Lancer l'analyse

Exécutez la commande suivante pour analyser votre documentation :

golink check --dir ./docs

Vous pouvez également vérifier un site web distant :

golink check --url https://votresite.com/docs

Étape 3: Analyser les résultats

GoLink générera un rapport détaillé. Pour comprendre ce rapport, consultez notre documentation sur le format de rapport (lien externe invalide).

Étape 4: Intégration avec CI/CD

Intégrez GoLink à votre pipeline CI/CD en suivant notre guide d'intégration CI/CD.
